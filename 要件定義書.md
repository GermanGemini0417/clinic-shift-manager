# 「Clinic Shift Manager for Front Desk」プロジェクト要件定義書

## 1. プロジェクト目的
本プロジェクトは、小規模クリニックの**医療事務・受付スタッフ**に特化したシフト作成・管理システムを構築することを目的とする。
クリニック特有の「午前・午後・夜間」の3部制において、スタッフのスキルセット（レセコン操作可否等）や休み希望を考慮した自動生成ロジックを実装し、管理者の工数削減と人員配置の最適化を実現する。

---

## 2. 機能要件

### 2.1 シフト管理・生成機能
* **スタッフマスター管理:** 氏名、職種（事務固定）、習熟度ランク（ベテラン/新人）、雇用形態、レセコン操作スキルの有無。
* **勤務枠定義:** 「午前診」「午後診」「夜間診」のコマ割り設定、および各枠の標準時間設定。
* **自動生成ロジック:** * スタッフが提出した「休み希望」を最優先で反映。
    * 各枠の必要最低人数を満たしつつ、必ず1名以上の「ベテラン」を配置する制約ロジック。
    * 特定スタッフの連続勤務日数や、特定時間帯（夜間診→翌午前診）のインターバル制限。
* **手動微調整:** 自動生成後のシフト表に対する、ドラッグ＆ドロップによる直感的なスタッフ入れ替え機能。

### 2.2 UI/UX機能
* **管理者用ダッシュボード:** 月間カレンダー形式のシフト一覧。欠員（必要人数未達）や制約違反（新人配置のみ等）を赤色で強調表示するアラート機能。
* **スタッフ用入力UI:** スマートフォンから自身の「休み希望」をカレンダー選択形式で登録・送信できるインターフェース。
* **確定通知:** シフト確定時に、スタッフ全員の画面へ反映および閲覧権限を付与。

### 2.3 システム・管理機能
* **権限管理:** 管理者（作成・編集・確定）と一般スタッフ（閲覧・休み申請）のロールベースアクセス制御。
* **データエクスポート:** 院内掲示用のPDF出力、および給与計算連携用のCSVデータ書き出し。

---

## 3. 非機能要件

### 3.1 パフォーマンス・互換性
* **マルチデバイス対応:** PC（管理者）、スマートフォン（スタッフ）の両方で最適化されたレスポンシブ表示。
* **リアルタイム性:** シフト修正内容が即座にプレビューに反映されるスムーズな操作感。

### 3.2 運用・保守
* **拡張性:** 将来的な「看護師」「技師」の追加を見越した、職種拡張が容易なデータ構造の設計。
* **CI/CD:** GitHub Actions等を用いた、迅速な機能追加とデプロイフローの構築。

---

## 4. 外部インターフェース仕様

### 4.1 データ構成
| カテゴリ | 項目 | 内容 |
| :--- | :--- | :--- |
| マスター | スタッフデータ | ID, 氏名, ランク, スキルタグ, 契約時間上限 |
| シフト | 枠定義 | 枠ID, 名称, 開始/終了時刻, 必要人数 |
| トランザクション | シフト結果 | 日付, 枠ID, 担当スタッフID, 確定フラグ |

### 4.2 操作体系
* **PC:** マウスによる一括割り当て、詳細設定、データ出力。
* **タッチ:** モバイル端末でのタップ・スワイプによる休み希望入力。

---

## 5. システムアーキテクチャ（MVP）

### 5.1 アーキテクチャフロー
1. **フロントエンド:** React / Next.js (Tailwind CSS)
2. **バックエンド:** Python (FastAPI) による最適化演算
3. **データベース:** PostgreSQL (スタッフ・シフト情報管理)
4. **デプロイ:** Firebase Hosting / Vercel 等

### 5.2 ソースカテゴリ
- `api/optimizer.py`: シフト自動生成の制約充足ロジック。
- `models/schema.py`: スタッフ、シフト、休み希望のDBスキーマ。
- `components/Calendar.tsx`: シフト表示・編集用カレンダーコンポーネント。
- `services/export_pdf.py`: 掲示用PDFの生成。
